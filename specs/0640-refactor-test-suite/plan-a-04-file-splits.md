## Step 1 — Add the oversized-test detector and seed the allow-list

Created: `scripts/check-oversized-tests.mjs`,
`scripts/check-oversized-tests.allow.json`,
`tests/check-oversized-tests.test.js`.
Modified: `package.json` (add `invariants:check-oversized-tests`
script and wire into the existing `invariants` aggregator).

**Order within the commit matters.** Author the script first, run
`node scripts/check-oversized-tests.mjs --seed >
scripts/check-oversized-tests.allow.json` to capture every file over
the ceiling on the branch HEAD (which equals `origin/main` at this
point), then wire the script into the `invariants` aggregator in
`package.json`. The same commit lands the script, the seeded allow-list,
the regression test, and the wiring — so `bun run check` exits zero on
the branch HEAD by construction.

```js
// scripts/check-oversized-tests.mjs
import { readFileSync, readdirSync, statSync, existsSync } from "node:fs";
import { join, resolve, relative, dirname } from "node:path";
import { fileURLToPath } from "node:url";

const ROOT = resolve(dirname(fileURLToPath(import.meta.url)), "..");
const CEILING = 400;
const SCOPES = ["libraries", "products", "services", "tests"];
const SKIP_DIRS = new Set(["node_modules", "dist", "generated", "tmp"]);
const SEED_MODE = process.argv.includes("--seed");

function loadAllow() {
  const p = join(ROOT, "scripts/check-oversized-tests.allow.json");
  if (!existsSync(p)) return new Set();
  const text = readFileSync(p, "utf8").trim();
  return new Set(text === "" ? [] : JSON.parse(text));
}

function walk(dir, out = []) {
  if (!existsSync(dir)) return out;
  for (const name of readdirSync(dir)) {
    if (SKIP_DIRS.has(name)) continue;
    const full = join(dir, name);
    if (statSync(full).isDirectory()) walk(full, out);
    else if (name.endsWith(".test.js")) out.push(full);
  }
  return out;
}

const ALLOW = loadAllow();
const violators = [];
const oversized = [];
for (const scope of SCOPES) {
  for (const file of walk(join(ROOT, scope))) {
    const lines = readFileSync(file, "utf8").split("\n").length;
    if (lines <= CEILING) continue;
    const rel = relative(ROOT, file);
    oversized.push(rel);
    if (ALLOW.has(rel)) continue;
    violators.push(`${rel}: ${lines} LOC > ${CEILING}`);
  }
}

if (SEED_MODE) {
  process.stdout.write(`${JSON.stringify(oversized.sort(), null, 2)}\n`);
  process.exit(0);
}

if (violators.length) {
  for (const v of violators) console.error(`error: ${v}`);
  process.exit(1);
}
```

The detector is intentionally narrow: a `wc -l` over the test glob and
an allow-list set difference. No AST parsing, no LOC normalisation. Per
[design Decision 5] this is the **regression** gate — existing oversized
files are seeded into the allow-list at the same commit, so the gate
catches new entries, not retroactively the current 30.

### Implementation-time seed procedure

At implementation time the seed list is generated by the script itself.
The implementer first commits the four splits (Steps 2–5), then runs:

```sh
node scripts/check-oversized-tests.mjs --seed > \
  scripts/check-oversized-tests.allow.json
```

This emits the over-ceiling set against branch HEAD as the allow-list,
so the detector exits zero against the same tree by construction. The
seed is **derived from `find`, not hand-typed** — any drift between
plan-time and implementation-time (driven by parts 02, 03, or 05's
LOC changes) is absorbed automatically.

A reference plan-time snapshot (for reviewer orientation only — the
implementer-time seed is the source of truth): 30 files over 400 LOC
on `origin/main` at plan time, including the four files this part
splits. After the splits land, the seed contains 26 entries.

The regression test asserts the detector flags a corpus file at 401
LOC and stays silent at 400 LOC.

Verification: after Steps 2–5 land and the seed is regenerated,
`bun run invariants:check-oversized-tests` exits zero on the branch
HEAD; any test file >400 LOC absent from the seed becomes a future
