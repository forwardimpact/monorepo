# Generate Code from Proto Definitions

`fit-codegen` reads the shared proto contracts and emits the generated code that
keeps JavaScript types, service endpoints, and MCP tools in sync with a single
source of truth. Every generated file carries a `@generated by fit-codegen`
header — do not edit generated files; change the proto and regenerate.

## When to Use

- Regenerate everything after a proto change — `npx fit-codegen --all`
- Generate one artifact kind — `--type`, `--service`, `--client`, `--definition`, `--metadata`
- Bootstrap a gRPC product for the first time

## Usage

```sh
# Generate all artifacts
npx fit-codegen --all

# Generate a single kind
npx fit-codegen --type        # JavaScript types only
npx fit-codegen --service     # Service base classes only
npx fit-codegen --client      # Typed clients only
npx fit-codegen --definition  # gRPC definitions only
npx fit-codegen --metadata    # Field metadata for MCP only
```

Combine flags to generate a subset: `npx fit-codegen --type --client` generates
types and clients only.
