gemba-trace 0.1.0 — Download, query, and analyze agent execution traces — read NDJSON output from gemba-harness as qualitative research

Usage: gemba-trace <command> [options]

Commands:
  runs [pattern]                             List recent GitHub Actions workflow runs (default pattern: kata|agent|eval|benchmark)
  find <run-id> <key>                        Resolve a trace lane for a known run id by key — exact filename, case, or participant; ambiguous keys error with the candidates
  download <run-id>                          Download a trace artifact and extract its .ndjson members (structured.json is produced only for single-member artifacts); pass --artifact to pick one when a workflow emits multiple `trace--*` artifacts
  overview                                   Metadata, summary, turn count, tool frequency
  count                                      Number of turns
  batch <file> <from> <to>                   Turns in range [from, to) (zero-indexed)
  head                                       First N turns (default 10; set with --lines)
  tail                                       Last N turns (default 10; set with --lines)
  search <file> <pattern>                    Search all content for regex pattern
  tools                                      Tool usage frequency (descending). See also `tool` (turns for one tool) and `tool-calls` (paired use+result records)
  tool <file> <name>                         All turns involving a specific tool. See also `tools` (frequency) and `tool-calls` (paired use+result records)
  tool-calls                                 One record per tool_use block, each paired with its tool_result by toolUseId (orphans emit result:null). See also `tool` and `tools`
  commands                                   One record per Bash tool_use block, carrying the command text
  paths                                      Distinct Read/Edit/Write file_path arguments, frequency-sorted
  compare <file-a> <file-b>                  Side-by-side comparison of two traces: turns, tools, paths, cost, and per-tool delta
  errors                                     Tool results with isError=true
  reasoning                                  Agent reasoning text only
  timeline                                   Compact one-line-per-turn overview
  stats                                      Token usage and cost breakdown
  cost <file>                                Total run cost across every participant (agent, supervisor, judge, named profiles), with a per-source breakdown
  init                                       Full system/init event
  turn <file> <index>                        Single turn by index
  by-discussion <discussion-id> [trace-dir]  List trace files whose meta header carries the given discussion_id, ordered by first-event timestamp
  filter                                     Filter turns by role, tool, or error status
  split <file>                               Split a combined trace into per-source files following the `trace--<case>--<participant>.<role>.ndjson` convention
  assert <test-name> <file>                  Shell-friendly assertion — outputs structured JSON for invariant hooks

Options:
  --help, -h         Show this help
  --version          Show version
  --json             Output help as JSON (use --format json for command output)
  --format=<string>  Command output format: text (default) or json
  --signatures       Include thinking.signature blobs in output

Examples:
  gemba-trace runs --lookback 7d
  gemba-trace runs --participant release-engineer
  gemba-trace find 27401632821 release-engineer
  gemba-trace find 27401632821 demo-task-r0
  gemba-trace download 24497273755
  gemba-trace split trace--default.raw.ndjson --mode=facilitate
  gemba-trace overview --file trace--default--agent.agent.ndjson
  gemba-trace overview --file trace--default--agent.agent.ndjson --format json
  gemba-trace timeline --file trace--default--agent.agent.ndjson
  gemba-trace stats --file trace--default--agent.agent.ndjson --by-tool
  gemba-trace cost trace--default.raw.ndjson
  gemba-trace cost trace--default.raw.ndjson --markdown
  gemba-trace tool trace--default--agent.agent.ndjson Conclude
  gemba-trace tool-calls --file trace--default--agent.agent.ndjson
  gemba-trace commands --file trace--default--agent.agent.ndjson --match '^git'
  gemba-trace paths --file 'traces/*.ndjson' --prefix /app
  gemba-trace compare trace-a.ndjson trace-b.ndjson
  gemba-trace search trace--default--agent.agent.ndjson 'error|fail' --context 1
  gemba-trace filter --file trace--default--agent.agent.ndjson --tool Bash --error
  gemba-trace turn trace--default--agent.agent.ndjson 3
  gemba-trace assert has-heading --grep '^## Problem' spec.md
  gemba-trace assert no-leak --not --grep 'password' output.log
  gemba-trace assert file-present --exists path/to/spec.md
  gemba-trace assert cites-jtbd --cites-job jtbd-excerpt.md spec.md
  gemba-trace assert used-edit --query "[?type=='assistant'].message.content[] | [?name=='Edit']" trace.ndjson
